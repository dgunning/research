## Tutorial Prompts

### Beginner Level (500-800 words)

1. **"Hello World Plus"**: Write a tutorial that goes one step beyond "Hello World" in your language
   - Time: 60 minutes
   - Include: Setup, code, common errors, next steps

2. **"Your First API Call"**: Teach someone to make their first HTTP request
   - Time: 60 minutes
   - Include: Different methods (GET, POST), error handling, parsing responses

3. **"Setting Up Your Dev Environment"**: Guide someone through setting up for your favorite language
   - Time: 90 minutes
   - Include: Installation, configuration, verification, first program

4. **"Git Basics for Beginners"**: Teach the 5 essential Git commands
   - Time: 60 minutes
   - Include: init, add, commit, push, pull with real examples

5. **"Creating Your First Function"**: Explain functions with a practical example
   - Time: 45 minutes
   - Include: Definition, parameters, return values, when to use them

### Intermediate Level (1000-1500 words)

6. **"Building a REST API in 30 Minutes"**: Create a simple but complete API
   - Time: 2 hours
   - Include: Routes, handlers, error handling, testing

7. **"Authentication from Scratch"**: Implement basic auth (API key or JWT)
   - Time: 2 hours
   - Include: Security considerations, implementation, testing

8. **"Database Integration Tutorial"**: Connect an app to a database
   - Time: 2 hours
   - Include: Connection, CRUD operations, error handling, best practices

9. **"Building a CLI Tool"**: Create a useful command-line utility
   - Time: 2 hours
   - Include: Argument parsing, error handling, help text, real-world example

10. **"Real-time Features with WebSockets"**: Add live updates to an application
    - Time: 2-3 hours
    - Include: Setup, connection management, message handling, fallbacks

### Advanced Level (1500-2000 words)

11. **"Microservices Communication"**: Implement service-to-service communication
    - Time: 3 hours
    - Include: REST, message queues, error handling, monitoring

12. **"Building a State Machine"**: Implement a finite state machine for a real use case
    - Time: 3 hours
    - Include: States, transitions, guards, real-world example

13. **"Implementing Caching Strategies"**: Add intelligent caching to an application
    - Time: 3 hours
    - Include: Different strategies, invalidation, monitoring, trade-offs

14. **"Writing a Custom Middleware"**: Create reusable middleware for your framework
    - Time: 2-3 hours
    - Include: Purpose, implementation, testing, composition

15. **"Building a Plugin System"**: Design and implement extensibility
    - Time: 3-4 hours
    - Include: Architecture, loading, lifecycle, security

## Deep Dive Prompts

### Intermediate Level (1200-1500 words)

16. **"How [Your Language] Handles Memory"**: Explain memory management in detail
    - Focus on: Allocation, garbage collection, or manual management

17. **"Inside [Framework Name]'s Router"**: Explain how routing actually works
    - Focus on: Pattern matching, middleware chain, performance

18. **"How Promises Really Work"**: Deep dive into async implementation
    - Focus on: Event loop, microtasks, implementation details

19. **"Database Indexes Explained"**: How indexes work internally
    - Focus on: B-trees, performance trade-offs, when to use them

20. **"The [Technology] Compilation Process"**: Explain compilation or interpretation
    - Focus on: Lexing, parsing, AST, optimization, bytecode

### Advanced Level (1500-2000 words)

21. **"How Git Stores Your Code"**: Explore Git's internal data structures
    - Focus on: Objects, commits, trees, blobs, references

22. **"Understanding React's Reconciliation"**: Explain the diffing algorithm
    - Focus on: Virtual DOM, fiber architecture, optimization

23. **"Database Transaction Isolation"**: Deep dive into ACID properties
    - Focus on: Isolation levels, phantom reads, implementation

24. **"How JWT Works Under the Hood"**: Explain structure and validation
    - Focus on: Encoding, signing, verification, security

25. **"Container Orchestration Internals"**: How Kubernetes schedules pods
    - Focus on: Scheduler, etcd, controllers, reconciliation loops

## Custom Prompt Generator

**Fill in the blanks to create your own prompt:**

"Write a [LENGTH]-word [TYPE] about [TOPIC] for [AUDIENCE] that focuses on [SKILL]."

**LENGTH**: 500, 800, 1200, 1500, 2000, 2500
**TYPE**: tutorial, deep dive, comparison, problem-solution, guide, review
**TOPIC**: [Your technical topic]
**AUDIENCE**: beginners, junior devs, mid-level devs, senior devs, managers
**SKILL**: clarity, structure, examples, voice, technical accuracy, engagement

**Example**:
"Write a 1200-word tutorial about building a REST API for beginners that focuses on clarity and complete code examples."

## Tips for Using These Prompts

1. **Don't overthink**: Pick one and start writing
2. **Set a timer**: Constraints boost creativity
3. **Write badly first**: Get words down, then revise
4. **Use the tools**: Run `readability-analyzer.py`
5. **Seek feedback**: Share your work
6. **Iterate**: Revise based on feedback
7. **Track progress**: Note what you learned
8. **Mix it up**: Vary types and difficulty
9. **Publish**: Real readers provide real feedback
10. **Reflect**: What worked? What didn't?

## Getting Unstuck

If you're stuck on a prompt:

1. **Talk it out**: Explain the concept verbally first
2. **Draw it**: Sketch diagrams before writing
3. **Find examples**: Look at how others explain it
4. **Start anywhere**: Don't start with the introduction
5. **Write badly**: Perfect is the enemy of done
6. **Take a break**: Walk away and come back
7. **Change prompts**: Try something easier
8. **Ask for help**: Discuss with others
